# 个人 Agent 定义、能力分层与可行方案（讨论稿）

## 一、Agent 的定义（从“工具”到“长期伙伴”）

建议定义为：

**与用户共享生活情境、持续积累共同经验、可被托付任务并能自我修正的个人智能伙伴。**

这个定义包含 4 个不可缺少的特征：

- 共享情境：它看到/听到/理解的世界，尽量接近用户正在经历的世界。
- 共同记忆：它记得的不是聊天记录，而是“共同发生过的事”。
- 托付能力：它不仅回答问题，还能把事做完，并对结果负责。
- 长期进化：它会因为用户反馈和生活变化调整自己，而不是永远停在初始版本。

## 二、面向用户需求的模块划分（8 大模块）

### 1) 共域感知模块（Shared Reality）

用户需求：  
我不想每次都解释背景；我希望它和我在同一个现场里。

能力分析：
- 空间认知：知道用户在哪、周边有什么、当前处于什么活动状态（通勤/开会/休息）。
- 实体与关系认知：识别人、物、地点、事件，并理解其关系（同事、家人、客户、常去地点）。
- 情境连续性：不是单帧识别，而是持续理解场景演化。
- 多模态融合：把语音、视觉、设备状态、日程信息合并成统一情境。

理想体验：  
它知道我刚从会议室出来，不用我重复背景就能接上刚才的话题。

### 2) 个体画像与人生记忆模块（Personal Memory）

用户需求：  
我希望它越来越懂我，而不是每次重新认识我。

能力分析：
- 偏好记忆：沟通风格、节奏、决策习惯、常用工具。
- 人生事件记忆：项目节点、关系变化、重要承诺、长期目标。
- 价值观与边界记忆：什么是用户在乎的，什么是不能越界的。
- 记忆分层：短期（今天）/中期（本周）/长期（多年）自动整理。
- **记忆生命周期**：时效与重要性综合加权（如 `lifespan_score`）、主动遗忘与用户「不重要」反馈、周期性相关性检视，避免陈旧记忆拖累检索。
- **查询沉淀（LLM Wiki）**：高质量问答可归档为新 Wiki 页，与「资料摄入」并列，形成知识复利。
- 可追溯记忆：用户可以追问“为什么这么判断”，系统可回溯依据。

理想体验：  
它不是记住我说过的话，而是记住我这个人。

### 3) 意图理解与协同对话模块（Intent & Dialogue）

用户需求：  
我说得模糊它也能懂；复杂任务它会和我对齐，不会盲目执行。

能力分析：
- 深层意图识别：区分情绪表达、信息查询、任务委托、价值冲突。
- 多轮澄清协商：关键节点主动问清楚，不关键节点尽量少打扰。
- **澄清决策树（与治理联动）**：将操作映射到分级 L0–L4（查询 / 内部写入 / 外部通信 / 财务·权限 / 他人关系），各级对应固定澄清策略（零确认、事后通知、草稿确认、强认证二次确认、暂停待授权）；**执行侧硬约束，不由模型单独改写**。
- 对话策略自适应：赶时间时简洁，需陪伴时细致。
- 跨场景承接：跨天、跨设备、跨应用保持同一任务线。

理想体验：  
它像一个懂分寸的人，而不是机械问答机。

### 4) 规划决策与执行编排模块（Planner）

用户需求：  
我只关心目标达成与风险，不想管每个步骤。

能力分析：
- 目标拆解：把模糊目标拆成可执行路径。
- 资源编排：调用应用、设备、服务、人际协作资源；**工具侧优先经 MCP（Model Context Protocol）接入**，复用社区 Server，专有 API 自建 MCP Server。
- 优先级与时机判断：决定现在做什么、延后什么、提醒什么。
- 不确定性处理：失败重试、备选方案、必要时升级请示。
- 结果闭环：完成后汇报、沉淀经验、更新后续计划。

理想体验：  
我只交代结果，它自行安排过程，并在关键决策点找我确认。

### 5) 社会关系与协作模块（Social Intelligence）

用户需求：  
它参与我的生活，就必须理解人际关系与协作语境。

能力分析：
- 关系图谱维护：谁和谁是什么关系、历史互动与敏感点。
- 社交语境判断：对老板、朋友、家人使用不同表达策略。
- 承诺与礼仪管理：跟进承诺、提醒致谢、避免失礼。
- 协作记忆：记录团队共识、分工、冲突点、决议演进。

理想体验：  
它不只帮我做事，也帮我把“和人协作”做得更好。

### 6) 情感陪伴与人格稳定模块（Companionship）

用户需求：  
我希望它有温度但不越界，长期稳定、可信赖。

能力分析：
- 情绪感知与回应：识别压力、疲惫、焦虑并给出合适回应。
- 人格一致性：语气、价值立场、行为风格长期稳定。
- 关系节奏感：知道何时提醒、何时安静、何时只倾听。
- 信任累积：靠持续靠谱与守边界建立长期依赖关系。

理想体验：  
它不是讨好我，而是在关键时刻让我更稳。

### 7) 自我进化与能力生长模块（Self-Evolution）

用户需求：  
我教一次，它下次就会；我纠正一次，它不再重复犯错。

能力分析：
- 经验提炼成技能：把成功流程固化为可复用能力。
- 失败反思机制：区分理解错、执行错、时机错。
- **负向验证**：新 Skill 配套反例用例，灰度期检测误触发，防止「偶然成功」放大为错误高置信技能。
- **多样性保持**：避免对单一用户过拟合；Skill 可多版本轮替，短期行为与长期目标冲突时可善意提醒（受治理策略约束）。
- 个体化学习：优先学习“对我有效”的行为模式。
- 进化可控：允许查看、回滚、冻结某类行为变化。
- 能力地图：知道自己会什么、不会什么、下一步学什么。

理想体验：  
它不是笼统“更聪明”，而是持续“更懂我且更靠谱”。

### 8) 安全、隐私与主权模块（Trust & Governance）

用户需求：  
它知道我很多事，但控制权必须始终在我手里。

能力分析：
- 数据分级与可见性控制：什么能记、记多久、谁可访问。
- 行为授权机制：高风险操作需要更高等级确认。
- 风险保护：涉及财务/法律/医疗/关系冲突时自动降速并请示。
- 可解释与可审计：关键决策可追踪、有记录、可追责。
- 紧急刹车：一键暂停与强制规则边界不可绕开。

理想体验：  
它能力很强，但不会失控替我做主。

## 三、能力分层：先攻克“底层能力”，再由底层反哺上层

你提出的方向非常准确：第一阶段不追求全面，而是先打牢可自我增长的底座。  
下面给出一版“底层 -> 中层 -> 上层”的依赖结构。

### A 层（底层能力，必须优先）

这些能力决定系统是否能“越用越好”，也是上层自然交互体验的前提。

1. 记忆与知识演化底座
- 长短期记忆分层、实体化记忆、事件时间线、检索与回溯。
- 记忆写入质量控制（去重、冲突检测、时效衰减）与 **检索侧生命周期**（如 `lifespan_score`、主动遗忘、相关性 Lint）。
- LLM Wiki：**资料摄入** 与 **查询沉淀** 双来源，形成知识复利。
- 可将高频记忆蒸馏成“低成本可调用能力”。

2. 自我进化引擎
- 从任务成败中自动提炼 Skill（成功模板/失败模式）。
- 反思闭环（计划偏差、执行偏差、反馈偏差）。
- **负向验证**（反例用例、灰度误触发监控）与 **多样性保持**，降低错误学习与单用户过拟合。
- 技能版本化与回滚，保证“可进化但可控”。

3. 意图理解与任务建模
- 从自然表达中抽取目标、约束、偏好、风险边界。
- 支持不完整指令下的主动澄清；**L0–L4 分级** 由治理层强制执行澄清策略。
- 意图可结构化存储，为后续自动化执行做输入。

4. 安全与主权治理
- 数据与操作分级、授权门控、审计日志；与 **澄清决策树** 对齐。
- 高风险动作的人在回路（Human-in-the-loop）。
- 策略执行不可绕过（即便模型“想走捷径”也不行）。

5. 统一执行总线（工具调用与状态管理）
- **优先 MCP**：工具发现、调用协议与社区生态对齐；专有接口通过 MCP Server 适配。
- 统一封装 API/设备/应用调用，记录调用上下文与结果。
- 支持失败重试、幂等、超时与补偿机制。
- 为学习系统提供标准化执行反馈数据。

6. 集成中枢（组件间胶水收敛）
- 多仓库/多协议组合（运行时、记忆、进化、治理、具身）经 **统一事件与适配器层** 交互，控制点对点调用失控；升级影响收敛到 Adapter。

### B 层（中层能力，由底层驱动生长）

- 共域感知（先基础，再增强空间/社交关系理解）。
- 规划与执行编排（从单任务到多任务并发与长期规划）。
- 协同对话（从问答到上下文协作）。

### C 层（上层体验能力，长期迭代）

- 情感陪伴与人格稳定。
- 社会关系深度协作。
- 全场景无缝代理（跨家庭/工作/出行等）。

> 关键逻辑：  
> 没有 A 层，B/C 层只能靠“演示效果”；  
> A 层做扎实后，B/C 层可以通过外部接入 + 自我学习持续增长。  
> **集成中枢 + MCP + 澄清决策树** 属于把 A 层从「能跑」推到「能长期维护」的关键拼图。

## 四、针对底层能力的可行方案（参考《个人agent助理.pdf》）

这里给出一版“可落地但不重”的方案骨架，核心目标是先构建可进化底座。

### 1) 底层总体架构（最小可行）

- 感知输入：Reachy Mini（或仿真）语音 + 视觉。
- 认知中枢：本地/云端 LLM（如 Qwen 系，多模态优先）。
- **集成中枢**：薄事件总线 + 强 Schema + 按组件 Adapter（首期可与记忆管线同步打通一条端到端路径）。
- 记忆底座：MemVerse（主）+ LLM Wiki（结构化知识层 + **查询沉淀**）。
- 执行框架：OpenClaw 或 Hermes 作为会话/编排外壳；**对外工具统一走 MCP Client**（社区 MCP Server + 自建专有 Server）。
- 进化机制：Leaper 类反思/技能提炼 + **反例灰度**（可先简化实现）。
- 治理层：策略引擎 + 审计日志 + 授权网关 + **L0–L4 澄清决策树硬约束**（自建轻量版）。

### 2) 对应到底层能力的“组件映射”

1. 记忆与知识演化
- 主框架：MemVerse（多模态、分层记忆、记忆蒸馏）。
- 结构化表达：LLM Wiki（`index + log + entity pages` 的 Markdown 体系）；**摄入与查询沉淀**均写入可追溯来源。
- **生命周期**：检索加权、主动遗忘与周期性相关性检视。
- 辅助人格记忆：SoulClaw（如需稳定人格与身份记忆）。

2. 自我进化引擎
- 主机制：Hermes/Leaper 的“任务后反思 -> Skill 生成 -> 下次优先调用”。
- 存储形式：Skill 文件版本化（含触发条件、步骤、风险、回滚）。
- **安全进化**：反例用例与灰度误触发监控；多版本/多样性策略减轻单用户过拟合。
- 学习反馈：用户纠错、任务成败、延迟与质量指标共同驱动。

3. 意图理解与任务建模
- 方案：LLM 输出标准化任务对象（goal/constraints/risk/confirm_level）并映射 **L0–L4**。
- 结合记忆：调用长期偏好与历史任务模板补全意图。
- 门控：**澄清策略由治理层按分级强制执行**，不仅依赖模型自觉。

4. 安全与主权治理
- 操作分级：读信息、发消息、改日程、转账等分级授权；与 **L0–L4** 澄清策略对齐。
- 数据分级：会话级、长期级、敏感级存储策略分离。
- 审计机制：每次关键动作记录“依据 -> 执行动作 -> 结果 -> 可回滚点”。

5. 统一执行总线
- **协议层**：MCP Client + 社区/自建 MCP Server；编排仍可由 OpenClaw/Hermes 驱动。
- 可靠性：统一超时、重试、熔断、补偿逻辑。
- 可学习性：统一事件日志格式，供记忆和进化模块消费。

6. 集成中枢
- 组件间经 Hub 交换标准事件/指令；版本升级在 Adapter 内吸收。

### 3) 实施优先顺序（只针对底层，不追求大而全）

Phase 1：记忆 + 安全 + 集成中枢骨架
- 建立事件日志与实体页（人/地点/项目/承诺）。
- 完成数据/操作分级与授权网关。
- **落地集成 Hub 最小骨架**（schema + 一条完整 Adapter 链路）。
- 打通“可查可追溯”的最小闭环。

Phase 2：意图理解 + MCP 执行总线 + 澄清决策树
- 构建标准任务对象与 **L0–L4** 确认机制（治理强制执行）。
- 经 MCP 接入 3~5 个高频工具（日历、邮件、文件、消息等，优先社区 Server）。
- 完成“理解 -> 执行 -> 反馈”闭环数据沉淀。

Phase 3：自我进化引擎 + 负向验证
- 上线任务后反思与 Skill 自动提炼；配套 **反例灰度**。
- 建立 Skill 版本控制、评估和回滚；关注多样性保持。
- 让系统在真实使用中每周稳定产生“可验证进步”。

Phase 4（可与 Phase 3 部分并行）：治理深化 + 记忆生命周期
- **lifespan_score**、用户主动降权、周期性记忆 Lint。
- Wiki **查询沉淀** 与 index 来源标注验收。

### 4) 可行性判断（为什么这条路可行）

- 与 PDF 的主张一致：先做记忆与进化闭环，而非追求全功能外观。
- 技术上有开源支点：MemVerse / OpenClaw / Hermes / SoulClaw / Reachy SDK 可组合；工具侧叠加 **MCP 生态** 降低重复造轮子。
- **集成中枢 + 强契约** 使多仓库组合可维护，避免集成债务在同一时期爆炸。
- 工程上可拆分：每层都有可独立验收的接口与指标；Phase 1～4 可按团队并行度调整，但 **Hub 骨架宜早**。
- 产品上可持续：底层能力一旦稳定，上层体验可通过插件与学习持续增长。

## 五、结论（用于当前决策）

你当前最重要的决策是正确的：  
**先定义“可进化底层能力”，再让其他能力逐步涌现。**

建议把“是否做到了以下要点”作为阶段验收：
- 记忆是否真实可用、可追溯，且具备 **生命周期与查询沉淀** 的演进路径。
- 进化是否可观察、可回滚、可验证，且 **负向验证** 覆盖新 Skill。
- 安全与主权是否始终先于自动化执行，且 **澄清决策树** 实际生效（非仅文案）。
- 工具与集成是否 **MCP 优先**、组件间是否经 **集成中枢** 收敛耦合。

当上述要点成立，这个 Agent 才有资格进入“全面融入生活”的下一阶段。
