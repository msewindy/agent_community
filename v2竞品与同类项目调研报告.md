# v2「共域陪伴版」竞品与同类项目调研报告

> **文档版本**：v1.0（2026-05-20）  
> **调研对象**：与 [`阶段A_需求驱动版设计.md`](./阶段A_需求驱动版设计.md) v2.0 定义相近的**已发布产品、开源项目、大厂路线**  
> **结论先行**：**目前没有一款产品/项目与 v2 完全同构**；最接近的是 **「Reachy 硬件 + 本地语音 Agent + 持久记忆」的拼装体**，或 **Meta Ray-Ban / Google Astra 的云端多模态路线**。v2 的真实机会在于：**本地优先 + 数据主权 + 开源可改 + 桌面具身 + Wiki 知识复利** 的组合，大厂闭源硬件赛道尚未覆盖这一交集。

## 2. 总体结论：有没有「完全一样」的项目？

### 2.1 直接答案

| 问题 | 答案 |
|------|------|
| 是否有**已商用、与 v2 十维全满**的产品？ | **没有** |
| 是否有**开源仓库、与 v2 十维全满**的实现？ | **没有**（多为 4–6 维子集拼装） |
| 是否有**单维做得极好、可借鉴**的？ | **很多**（见 §4–§7） |
| v2 最大竞品威胁来自哪里？ | **Meta（眼镜+记忆收购）**、**Google（Astra/Gemini Live）**、**Apple（2027 吊坠+情境 Siri）**——均为闭源、云端、生态锁定 |
| v2 最大差异化空间？ | **本地 GPU 全栈 + Reachy 开源具身 + Wiki/MCP/治理可审计** |

## 8. v2 相对竞品的「组合独特性」

用集合表示，**已发布最接近 v2 的并集**约为：

```
Reachy Conversation App
  ∪ OpenClaw(记忆+主动+Wiki插件)
  ∪ Hermes(MCP+Skill)
  ∪ A.T.O.M(本地语音视觉)
  ∪ M3-Agent(多模态记忆研究)
  ∪ Karpathy Wiki 社区版
```

**仍缺少 v2 同时强调的**：

1. **本地优先默认路径**（非 Realtime API 默认）  
2. **MemVerse 级记忆 + supersede + 记忆面板**  
3. **L0–L2 治理 + 校准话术**  
4. **Wiki 查询沉淀闭环**（竞品 Wiki 均无语音触发沉淀）  
5. **Reachy 具身 + 未来眼镜第一人称** 的统一记忆 schema（M3-Agent 仅研究）

→ 这不是「做一个新 ChatGPT」，而是 **「开源可改的、有身体的、会攒知识的本地 Jarvis」**——**品类上仍空白**。

## 12. 一句话战略结论

> **v2 没有正面竞品，但有大量「零件级冠军」。** 成功路径 = **用 Reachy 解决「身体在场」+ 用 OpenClaw/Hermes 解决「大脑与工具」+ 用 MemVerse/MCP 解决「记忆与执行」+ 用 Karpathy Wiki 解决「知识复利」+ 用自研薄层解决「本地、治理、校准、数据主权」**——避免做第二个 Friend，也避免做第二个 Humane。
