## 1. 背景（2026-07-06 家庭 Alpha 复盘）

沪教三年级英语 PDF 入库后，曾出现：

- 知识点直写 catalog（正确）与 **38 道待审习题误入知识点入库**（错误）并存；
- 经 **流程修复**（非一次性补丁）后：待审习题进 `question_inbox` → 家长端 **习题处理 → 待归类**；知识点入库仅保留含 KP 变更的批次。

家长确认页面显示正常后，提出三个前瞻问题：

1. 以后再跑 PDF 录入，是否仍会得到正确分流？
2. 修改是一次性修数据，还是入库流程优化？
3. 若增加「家长页 + 大模型对话」录入，能否复用上述能力？

## 2. 已落地的流程优化（后续 PDF 应沿用）

| 能力 | 路径 | 说明 |
|------|------|------|
| 沪教 PDF 脚本 | `scripts/ingest_hujiao_g3_english.py` → `run_hujiao_g3_english_ingest()` | KP/wiki 直写；可判分题进题库；待人工题进 inbox |
| 待审习题队列 | `student_data/_question_inbox/pending.json` | `QuestionInboxService` |
| 家长待归类 UI | `/exercises` 待归类 | 拍照待归类 + 课本待审（同页） |
| 知识点入库过滤 | `/kp-review` | 默认不列「仅练习题」批次 |
| 一次性迁移 | `scripts/migrate_question_jobs_to_inbox.py` | 仅清历史脏 job；新环境不必再跑 |

**结论**：同一沪教脚本 + 已部署代码 → **应与 2026-07-06 正确结果一致**；非沪教教材需新解析器，但分流原则可复用。

## 3. 待办项（产品 / 工程）

### T1 · 家长页 PDF 一键入库（未做）

- **现状**：PDF 入库仅 CLI；家长页无上传 PDF 入口。
- **目标**：家长选 PDF → 后台调解析器 → 走与 CLI 相同的 catalog / 题库 / inbox 分流。
- **依赖**：教材专用解析器（当前仅 `hujiao_g3_english_parser`）；WSL/Windows 环境一致（如 `pymupdf`）。

### T2 · 家长页「对话式录入」（未做）

- **现状**：无聊天 UI；无「对话 → 结构化草稿」编排层。
- **应复用（勿重写）**：
  - `QuestionInboxService.upsert_from_draft()` — 待人工题
  - `import_draft_questions()` — 可自动判分题
  - `KpCatalogWriter` / `merge_approved_draft` — KP 直写或可选 kp-review
  - `KpIngestReviewService` — 若要求家长批准 KP 变更
  - `/exercises` 待归类 — 展示与导入，**不必另开页面**
- **需新建**：对话 UI、会话状态、LLM 编排（输出 `KpDocumentDraft` 或分步指令）、上传 PDF/图片入口。
- **禁止**：待审习题再 `submit_kp_document()` 进知识点入库。

### T3 · 新教材 / 新学科 PDF 解析器（按需）

- 复制「解析器 + ingest + 三分流」模式；共用 inbox 与家长端。
- 与 [问题4](./问题4-课本多模态题型分析.md) 联动：多模态题型能力未齐前，解析器应 **显式标记**「仅课堂练 / 不推题」。

### T4 · 38 道 inbox 占位题

- **状态**：✅ 已清理（`scripts/drop_classroom_inbox.py`）；再入库走 `classroom_activities.json`
