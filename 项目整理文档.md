# agent_community 项目梳理与整理方案

> **文档版本**：v1.0（2026-06-02）  
> **目标**：梳理当前仓库构成；明确「记忆 + 自我进化智能 Agent」核心与可归档内容；为后续迁入 `temp/` 提供依据。  
> **说明**：本文档仅做分析与规划，**尚未执行任何文件移动**。

---

## 1. 项目定位（一句话）

基于 **Hermes Agent** 运行时壳 + **MemVerse** 记忆后端，按 M0–M8 里程碑实现的「共域陪伴型个人智能体」工程仓库——设计文档、自研薄层代码、18 个开源参考仓库与运行数据混存于同一根目录。

| `perception_data/` | 5 | <1 MB | 感知事件运行数据 |
| `calibration_data/` | 1 | <1 MB | 校准事件日志 |
| `tools_data/` | 2 | <1 MB | MCP 工具沙箱数据 |

## 5. 能力地图：设计目标 vs 代码现状

来自 `个人智能体能力清单与理论成熟度评估.md` 的 12 项能力，映射到本仓库实现状态：

| 能力 ID | 名称 | 成熟度 | 本仓库状态 | 模块 |
|---------|------|--------|-----------|------|
| **C1** | 跨会话持久记忆 | ★★★ | **已实现骨架** | `memory/` + MemVerse |
| C2 | 知识沉淀（LLM Wiki） | ★★ | 已实现骨架 | `wiki/` |
| C3 | 工具调用（MCP） | ★★★★ | 已实现 | `tools/` |
| C4 | 意图理解 + 澄清 | ★★ | 依赖 Hermes | hermes-agent |
| C5 | 安全主权与治理 | ★★★ | 部分（Gate + L0–L2） | memory/gate, tools |
| C6 | 集成中枢与可观测 | ★★★ | 已实现 | audit, trace_id |
| **C7** | **自我进化 / 技能学习** | ★★ | **v1 明确不做；仅调研** | leaper/reflexion/Voyager 在 research_repos |
| C8 | 任务规划 | ★★★ | Hermes 内置 | hermes-agent |
| C9 | 共域感知 | ★★ | M4 骨架 | perception/ |
| C10–C12 | 社会/情感/时机 | ★ | 未实现 | proactive 仅 L0/L1 骨架 |

**与你的关注点对齐**：

1. **记忆（C1）**：已有可运行路径 — `memory_service` → `MemVerseAdapter` → Docker MemVerse；Hermes 插件 `agent_memverse`。
2. **自我进化（C7）**：设计文档将进化推到 **v2/阶段 C**；参考实现已 clone（Leaper、Reflexion、Voyager、MemOS Reflect2Skill），但 **`agent_platform/` 内尚无 evolution 模块**。

## 9. 风险与注意事项

1. **路径耦合**：多个 Hermes 插件内含 `AGENT_COMMUNITY_ROOT` 文件，移动目录后需批量更新。
2. **体积**：即使移走 4 个 research repo，仍可能 >1 GB；可考虑 git submodule 或「按需 clone 清单 + 脚本」替代全量入库。
3. **验收脚本**：`agent_platform/README.md` 中大量 smoke/accept 命令跨 M1–M8；精简后需重写最小验收集（memory + hermes 工具）。
4. **Windows 环境**：README 示例路径为 Linux（`/home/lfw/...`）；在 Windows 上需调整 `PYTHONPATH` 与 Docker 路径。
